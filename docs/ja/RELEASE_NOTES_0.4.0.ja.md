# リリースノート v0.4.0 (2026-03-09)

## 概要
本リリースは、v0.3.0 で広がった機能を土台に、ローカル限定ドラッグ&ドロップの正式採用、キー操作の安定化、品質ゲートの強化を中心に整理した更新です。

## 主な変更
- ローカル限定ドラッグ&ドロップを正式採用
  - `Explorer -> ReflexFiles` のドラッグ取り込み
  - `ReflexFiles -> Explorer` の `Ctrl+Alt + 左クリック` によるドラッグ転送（ローカル限定、コピーのみ）
  - Google Drive (`gdrive://`) のドラッグ&ドロップは対象外を維持
- キー操作 / PATH補完の品質改善
  - 主要ショートカットの回帰検出を強化
  - モーダル/オーバーレイ中のショートカット漏れを修正
  - PATH補完の候補表示・取消・ステータス文言を改善
  - PATH補完中の `Enter` の挙動を仕様として固定し、文書化
- 品質 / リリース運用の改善
  - 軽量ドラッグ&ドロップ回帰テスト `npm run test:dnd` を CI の `quality` に追加
  - smoke E2E で `Ctrl+Shift+N` / `Ctrl+F` / `F2` / ZIP操作 / URL追加を確認
  - `npm audit --audit-level=high` が通るように lockfile を更新

## ドラッグ&ドロップの対応範囲
- 対応:
  - Explorer から ReflexFiles へのローカル項目のドラッグ取り込み
  - ReflexFiles から Explorer へのローカル項目のドラッグ転送（`Ctrl+Alt + 左クリック`）
- 非対応:
  - Google Drive (`gdrive://`) のドラッグ&ドロップ
  - 書き出し時の move 動作（正式仕様はコピーのみ）

## キー操作 / PATH補完の安定性
- モーダルやオーバーレイが開いているときのショートカット漏れを修正しました。
- テキスト入力欄では `Ctrl+C/X/V` が通常どおり動作するよう維持しました。
- PATH補完の表示を見やすく調整しました。
  - 候補件数の文言短縮
  - 候補なし専用メッセージ
  - `Esc` 取消時の明示メッセージ
  - 候補表示中に状態表示が戻りやすいよう調整

## セキュリティと依存関係
- `npm audit --audit-level=high` が通るよう、フロントエンド依存の解決結果を更新しました。
- Google Drive readonly stub 構成でもビルドが通るように feature 境界の不整合を修正しました。
