# リリースノート v0.3.0 (2026-02-22)

## 概要
本リリースは、v0.2.0（ビューア統合）を土台に、Google Drive の段階実装、キー操作の復旧・改善、リリース運用の強化を中心に進めた更新です。

## 主な変更
- Google Drive 対応（段階実装）
  - ユーザー自身の Google Cloud / OAuth 設定フロー
  - 実 API / スタブ backend の切り替え
  - `gdrive://` 一覧取得と対応形式の閲覧
  - ローカル作業コピーを使う外部アプリ連携
  - 手動の Google Drive 書き戻し
  - 競合時の検知と対処ガイド
- キー操作・操作性の改善
  - `Ctrl+N` = 新規ファイル作成
  - `Ctrl+Shift+N` = 新規フォルダ作成
  - `Ctrl+F` / `F2` / `Ctrl+Alt+Z` / `Ctrl+Alt+X` の復旧・強化
  - PATH補完の候補循環・確定挙動の改善
  - ZIP解凍ダイアログをキーボード操作しやすいUIに改善（衝突時のみ確認）
- 品質 / 開発運用の改善
  - 軽量キー回帰テスト `npm run test:keys` を追加
  - CI の `quality` にキー回帰チェックを追加
  - `e2e:full` は手動回帰テストとして運用（PR必須ゲートにしない）

## ドキュメントと公開方針の更新
- 公開ドキュメント（`docs/`）と内部作業記録（`development_documents/`）を明確に分離しました。
- ADR / Google Drive 脅威モデル / Google Drive 自己設定メモ / 生成キー操作一覧は、内部管理（リリース対象外）文書として扱います。
- ユーザー向けの Google Drive 設定・利用手順は、ユーザーマニュアル（EN/JA）へ直接統合しました。

## セキュリティ・資格情報の扱い
- 公開リポジトリに Google API 資格情報を含めない運用を維持（公開文書はプレースホルダのみ）。
- Google Drive のクライアントシークレット / トークン等はローカル管理前提として整理。
- リポジトリ健全性向上のため、`app/.npm-cache` を Git 管理対象から除外。

## 品質・安定性
- Tauri/WebView 経路で発生していたショートカット・フォーカス不具合を修正。
- モーダル確定の二重実行により一瞬誤表示される作成エラーを修正。
- E2E 実行時のポート/プロセス再利用に関する運用安定性を改善。
